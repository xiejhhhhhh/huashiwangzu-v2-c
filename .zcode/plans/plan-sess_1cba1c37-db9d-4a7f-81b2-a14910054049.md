## 加速计划：四个方向一次性落地

### 1. 一键诊断工具（diagnose MCP tool）
**新文件**：`dev_toolkit/diagnose_tools.py`
**模式**：复用现有 TOOL_NAMES + tool_definitions() + handles_tool() + handle_tool() 模式

**新增工具：`diagnose`**
在单次调用中聚合以下步骤返回结构化诊断报告：
- 后端健康检测（probe GET /api/health）
- 数据库连接检测（SELECT 1）
- 系统资源快照
- 按关键词/模块聚合错误日志
- 指定模块日志尾部

**接线文件**：`dev_toolkit/server.py`（添加 import 和 list_tools/call_tool 注册）

### 2. 增加编辑配方
**文件**：`dev_toolkit/edit_tools.py`
在 recipe_catalog() 和 build_recipe_operations() 中追加 3 个新配方：

| 配方名 | 功能 |
|--------|------|
| add_import | 文件顶部追加 import（跳过已有） |
| wrap_in_try_except | 把 old_text 包进 try/except |
| replace_in_file | 整个文件中替换字符串（允许替换多处） |

### 3. 省掉计划模式（行为约定）
对于明确、范围小、不影响架构的修改（改 1-3 个文件、不新增 API、不改 manifest）直接执行，不进计划模式。涉及跨模块契约/新增能力/架构决策时再进入计划模式。

### 4. 并行子代理模板
排查问题时默认并行模式：
- API 报错：bug_logs + code_explore(错误符号) + probe(出错API)
- Agent 异常：agent_runtime_snapshot + tail_log(agent) + bug_logs(agent)
- 前端问题：bug_logs(frontend) + 前端目录扫描
