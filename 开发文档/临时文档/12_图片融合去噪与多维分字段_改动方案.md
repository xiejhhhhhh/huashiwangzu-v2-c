# 12 图片融合去噪 + 多维分字段(第四层改动方案)

## 问题(已核实)
图片 round1(source_type=text,实为本地像素分析)= "本地图片分析:JPEG 4032x2268px RGB
平均亮度0.53 边缘密度…",被 `_llm_fuse` 当"第1轮文本提取"喂进 LLM、揉进 `fused_text`;
`_fallback_fusion` 挑第一个非空 round,图片 OCR(round2)空时挑中 round1 噪声。
→ fused_text 混像素噪声,对 agent 回答零价值,占召回权重、稀释命中。

非图片(PDF)round1 是真提取文本(如"hello smoke"),必须保留。

## 现状(好消息:字段全现成,分维度已在做)
`kb_page_fusions` 已有:fused_text / page_summary / page_title / body_json(实体) /
attributes_json(属性) / tags_json(标签)。`_llm_fuse` 已输出这些维度。
问题只在:图片 round1 噪声混入 fused_text + 兜底挑噪声。

## 改动(图片专属分支,不碰 PDF 逻辑,爆炸半径可控)
文件:`modules/knowledge/backend/services/fusion_service.py`

1. 顶部引入 `from .parsing_service import IMAGE_FORMATS`(现成常量,不新增)
2. `fuse_all_pages` 读 doc 后算 `is_image = (doc.extension or '').lower().lstrip('.') in IMAGE_FORMATS`,
   传给 `_run_page` → `fuse_page`
3. `fuse_page(is_image=...)`:
   - 图片:把 round1(像素分析)从"喂融合的 round_texts"里剔除,单独存进 attributes_json
     的 `图像技术属性` 字段;融合只用 round2(OCR)+round3(VLM)
   - 非图片:round_texts 原样(含 round1 真文本)
4. `_llm_fuse(round_texts, is_image=...)`:图片时 user_message 不再有"第1轮文本提取",
   改成"第1轮OCR文字 / 第2轮视觉描述"两块;prompt 明确"技术元数据不进正文"
5. `_fallback_fusion(round_texts, is_image=...)`:图片兜底只从 OCR/VLM 挑,永不挑 round1 像素噪声

## 多维分字段(agent 秒定位图,不实时跑 VLM)
融合已分维度,本次确保图片场景填满:
- OCR文字(图里的字)→ fused_text 主体(可被语义打齐纠错)
- VLM画面描述 → page_summary / body_json
- VLM标签+素材类型 → tags_json(搜"会议现场"命中)
- 像素技术元数据 → attributes_json.图像技术属性(不进召回)
效果:搜画面内容命中标签、搜图里公司名命中OCR文字 → 秒级定位,永不实时跑VLM。

## 回退路径
- 改动集中在 fusion_service.py 3个函数,加 is_image 分支,不删原逻辑
- git 分支保留,出问题 `git revert` 单个 commit 即回退
- 存量不动:改完先只影响新进文档;存量图片需重跑 fusion 才生效(force_fusion),
  分批重跑、可中断续跑,不一次性全刷

## 风险
- prompt TFUSION 若硬编码"三轮"措辞,需同步调整(存 MySQL/DB,动态加载,确认后改)
- 非图片路径必须零改动(回归验证:PDF 融合前后 fused_text 一致)
