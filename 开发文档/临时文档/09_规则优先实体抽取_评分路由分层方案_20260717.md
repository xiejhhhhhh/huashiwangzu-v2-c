# 规则优先实体抽取 · 评分路由分层方案

> 日期:2026-07-17　准备期
> 起因:图谱阶段"每页喂LLM"抽实体,710页文档烧706次LLM,还整页抽空(doc4275第7页26个成分全丢)。华哥定方向:规则/词典当主力,LLM只补漏,抽完打分、分不够才拉模型。

## 一、问题诊断(已用生产数据坐实)

### 现状链路
`kb_raw_data(原始OCR/VLM) → kb_page_fusions(按页融合) → 每页喂LLM抽实体 → 去重写图谱`

核心代码:`modules/knowledge/backend/services/entity_service.py:514` `_extract_page()`
- 全流程唯一门槛:`len(text) < 20` 才跳过,其余每页无脑烧一次LLM
- 所有页 `asyncio.gather` 并发烧

### 坐实的毛病(doc4275 苏蜜雅安全评估报告,710页)
| 指标 | 数值 |
|---|---|
| 总页 | 710 |
| 烧LLM页(≥20字) | 706 |
| 抽出实体 | 1094 |
| 有产出的页 | 292 |
| **0产出的页** | **414(占59%)** |

- 第7页"表2成分总含量表"(丙二醇62.63%→甘草亭酸0.0024%,共26成分)**整页0产出**
- 探针验证:同页同提示词,现在打deepseek **8.9秒抽全26个成分,完美**
- **结论:不是提示词/模型能力问题,是那次LLM调用返空被静默吞掉,无重试无校验无兜底。一次偶发失败,26个核心成分永久丢失。**

### 更深一层(华哥点破)
"每页喂LLM"本身就是后期加工、方向反了。成分表"序号+名称+INCI+含量%"这种格式固定的,**正则一扫就出,根本不该烧LLM**。LLM该只在"需要理解"的地方兜底。

## 二、目标架构:四层刷 + 评分路由

原则:**能规则的先规则,一层层往下刷,抽过的进词典下次不重抽;每层抽完打分,分够收工、分不够才拉模型。**

```
┌─ 第一层 正则硬抽(0 LLM) ──────────────┐
│  格式固定的直接扫:成分/INCI、规格、备案号、  │
│  日期、金额、电话、机构后缀(有限公司/研究所) │
└───────────────┬───────────────────┘
                ↓ 未覆盖文本
┌─ 第二层 词典命中(0 LLM) ──────────────┐
│  已积累实体词典 → pg_trgm模糊匹配整篇文案   │
│  命中直接标注。词典越大越快、越省          │
└───────────────┬───────────────────┘
                ↓ 未覆盖文本
┌─ 第三层 分词缩句抽新词(0 LLM或轻量) ─────┐
│  分词+词性,留名词性主体做"候选新词"进池    │
│  (jieba / PG zhparser)                 │
└───────────────┬───────────────────┘
                ↓ 候选新词 + 覆盖率评分
┌─ 评分路由 ────────────────────────────┐
│  给这一页/这一块打"规则覆盖分":          │
│   高分 → 规则结果直接收,不烧LLM          │
│   低分 → 拉LLM补抽(只喂这页+已抽实体清单) │
└───────────────┬───────────────────┘
                ↓ 仅低分页
┌─ 第四层 LLM补漏 ─────────────────────┐
│  不喂全文710页,只喂低分页/新词批         │
│  判类型、消歧、连关系                    │
└──────────────────────────────────────┘
```

### 评分怎么打(核心)
一页/一块的"规则覆盖分" = 综合:
- **命中密度**:规则+词典抽出的实体数 / 该页名词性词数(缩句后)
- **格式置信**:是否命中强格式(表格行/成分行/备案号→高置信)
- **残余可疑**:分词后有多少"未被任何规则/词典命中的名词性主体"(残余越多分越低)

分数阈值(初定,探针后校准):
- ≥ 0.8:规则结果直接收,0 LLM
- 0.5~0.8:规则结果 + LLM轻量复核(只喂候选新词,不喂全文)
- < 0.5:拉LLM全量抽这一页

## 四、哪些能纯规则、哪些必须LLM

### 纯规则可搞定(0 LLM)
- 成分:INCI标准词表(公开可导入)+ "序号 中文名 INCI 含量%" 行模式
- 规格:正则 `\d+(\.\d+)?\s*(%|克|g|ml|毫升|mg)`
- 备案号:`国妆[网]?备字?\w+` / `粤G妆网备字\w+`
- 机构:后缀 `(有限公司|研究所|检测中心|监督管理局|协会)`
- 日期/金额/电话/地址:标准正则

### 必须LLM(补漏)
- 关系抽取:A属于B、A产自B(共现能给候选,判定要LLM)
- 语义归类:新词是成分还是品牌还是术语
- 歧义消解:同名不同义
- 隐含实体:话术类文案里"我们的王牌爆品"指代谁

## 八、实体挖掘层定稿:三道关噪音过滤(越用越省)

探针D坐实:大量噪音规则就能干掉,不用等LLM。实体挖掘 = **三道关,越往后越贵、量越少**。

```
分词结果(kb_terms,已落盘68万词)
        ↓
┌─ 第①关 规则黑名单(0 LLM,当场丢) ──────────┐
│  · VLM模板碎片:以"为/约/包括/如下"结尾的描述句  │
│  · 技术噪音:纯色值(ffffff)/纯尺寸(x3448)/单位(ml)│
│  · 停用词:因此/所以/例如/包括/围绕            │
│  · 长句误切:字数>12且含谓语,判句子不是词      │
│  · 英文通用词表:BUSINESS/MARKETING/CREATE    │
└──────────────┬────────────────────────┘
               ↓ 未被黑名单命中
┌─ 第②关 规则白名单(0 LLM,当场留) ──────────┐
│  · 命中已有实体词典/成分INCI表 → 直接留        │
│  · 机构后缀(有限公司/研究所/检测中心) → 机构   │
│  · 强格式正则(备案号/规格/日期) → 对应类型     │
└──────────────┬────────────────────────┘
               ↓ 前两关都没盖住的"疑难词"
┌─ 第③关 LLM精判(花钱,量已很小) ───────────┐
│  · 带一句上下文(不喂孤词!)                  │
│  · 判:是实体还是噪音 / 是什么类型            │
│  · 结果回写黑名单/白名单/词典                 │
└──────────────────────────────────────────┘
               ↓
        回写词典 → 下次同词被①②关拦下,不再喂LLM
```

**越用越省的机制**:第③关LLM判出的结果,回写进黑/白名单和词典。下一轮同样的词直接被前两关拦截,不再花LLM。**刷一轮规则厚一层,LLM的活少一批,最终趋近于0。**

### 三道关各自的量(基于探针D的91%噪音率外推)
- 第①关黑名单:预计干掉 63.7万里的大头(模板碎片+虚词+长句误切),0 LLM
- 第②关白名单:命中成分表/机构/格式,0 LLM
- 第③关LLM:只剩~3.6万疑难词批处理,≈450次LLM(带上下文)

## 九、真病根:根本没有分词器(2026-07-17 深挖坐实)

前面第①~③关都在"筛词",但深挖发现更底层的问题:**输入的根本不是词,是整句**。再好的筛选/LLM都是在错误的输入上打转 —— 这才是华哥说的"舍本逐末"。

### 病根:一行贪婪正则(cognitive_index_service.py:41)
```python
_TOKEN_RE = re.compile(r"[一-鿿]{2,}|[A-Za-z][A-Za-z0-9_+-]{1,}|[A-Z]{1,8}\d{3,}")
```
`[一-鿿]{2,}` = 抓2个以上连续中文,贪婪。**一串中文中间没标点/空格,整句全被抓成一个"词"**。配 extract_terms() 里区区7个停用词裸跑。

**所以"分词层"从头到尾没有分词器,只有这一个正则把连续中文抓成一坨。**

### 实证(全库真实数据)
- 68万"词"里 18.3万是 >12字的整句/半句(如"多数人通常都不会把自己的要求很明确地说出来"当成一个词)
- 喂给 grok/deepseek 的不是词是句子 → 模型判"这句是什么类型",本来就是错题
- 之前一路想加 LLM 补(重试/换grok/放量),全是在漏底的桶里灌水

### 解药链(逐层实证通过)
```
① jieba分词 + 业务词典(kb_entity_dictionary 10.5万词 + 成分表 + 通名表 灌进jieba)
   实证:娇薇诗/臻肌焕颜/精华液 专名不碎;广州娇宇化妆品/有限公司 主体通名分开
② 双向剥离:剥头部噪音(XX为/包括/本页为/送检单位为) + 剥尾部通名(有限公司/精华液)
   实证:送检单位为广州娇宇化妆品有限公司 → 主体『广州娇宇化妆品』
③ 通名后缀表 → 主体定类目(0 LLM,结尾自己写答案:有限公司→机构,精华液→产品,提取物→成分)
④ 剥完仍带谓语/超长 → 判句子碎片,踢出词表(不进 kb_terms)
⑤ LLM只兜底"剥完拿不准的纯专名",量极小,甚至先不用
```

### 单字通名的雷(必须多字或带主体约束)
单字后缀裸用会误伤:素(因素/像素/激素)、水(薪水/口水)、酸(丙二酸对但要看)、油(控油)、斑(色斑对但光斑不对)、肌(敏感肌对)。**只用多字通名(有限公司/精华液/提取物/检验报告)100%安全,单字必须前接已知词根或配黑名单。**
