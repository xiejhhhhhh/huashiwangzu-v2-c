# 规则优先实体抽取（待落地）

> 日期：2026-07-18
> 状态：**未落地**
> 来源：蒸馏临时文档 09；只保留目标架构与验收，不保留施工流水。

## 1. 问题

图谱阶段“每页喂 LLM”：

- 长文档页数 ≈ LLM 次数，成本高
- 偶发空响应被吞 → 整页实体永久丢失
- 固定表格（成分/INCI/规格）本可用规则抽出

证据级案例（历史）：710 页文档烧 706 次，59% 页 0 产出；同页探针可一次抽全。

## 2. 目标架构

原则：**规则/词典主力，LLM 只补漏；抽完打分，分不够才拉模型。**

```text
L1 正则硬抽（0 LLM）
  → 成分/INCI、规格、备案号、日期、金额、机构后缀等
L2 词典命中（0 LLM）
  → 已积累实体词典 + 模糊匹配
L3 分词候选（0/轻量）
  → 名词性主体进候选池
评分路由
  → 高覆盖：直接收；低覆盖：仅低分页进 L4
L4 LLM 补漏
  → 只喂低分页 + 已抽清单，做类型/消歧/关系
```

抽出结果回写词典，越用越省。

## 3. 落点预期

- 主改：`modules/knowledge/backend/services/entity_service.py` 及图谱阶段调用链
- 词典/规则：模块内独立表或配置，禁止假绿
- 与网关关系：LLM 补漏仍走统一 gateway；预算不足 deferred

## 4. 验收

1. 固定格式页：0 LLM 或极少 LLM 抽出关键字段
2. 低分页才触发 LLM；总调用量相对“每页一烧”明显下降
3. 空响应有重试/校验，不再静默丢整页
4. 抽样实体准确率不低于当前好结果基线
