# 华世王镞 V2 开发文档（总索引）

> 本文件是**总索引**，只讲全貌 + 指路，细节在各子文档里，不重复。新会话先读这里。

## 项目目标

Python FastAPI 后端 + Vue 3 桌面壳 + 可热插拔业务模块，给 20-50 人内部用的企业业务助手（不是 coding 平台；面向化妆品/美业集团）。

```text
frontend/   桌面壳前端（登录、桌面、窗口、任务栏、启动器、模块加载）
backend/    平台服务层（API、数据库、权限、任务、模型网关、文件存储）
modules/    业务模块（sandbox 独立开发 → manifest 接入）
```

## 当前状态（速览，细节见子文档）

| 分类 | 现状 |
|------|------|
| 前端 | Vue 3 + Vite(Rolldown) + Pinia，`vue-tsc` 0 错误 |
| 后端 | FastAPI + SQLAlchemy async，端口 33000，`--workers 3`，模型网关含降级/视觉 |
| 数据库 | PostgreSQL；`framework_*` 框架表 + 各模块 `{key}_*` 表（owner 隔离、无外键） |
| 模块 | **已接入 ~26 个**，见下「模块地图」 |
| 知识库 | **五层多轮已落地**（原始三轮→融合→画像→图谱→跨文件关联），见 `modules/knowledge` |
| Agent 引擎 | **上下文装配引擎建设中**：批1事件溯源+动态预算✅ 批2分层记忆✅ 批3经验记忆✅ 批4韧性/批5可观测进行中。设计见邮箱《引擎架构总纲》，调优见 `算法调优手册.md` |

## 模块地图（细节看各模块自己的 README）

- **核心**：`agent`（AI 助手，对话/SSE/渐进式工具发现/三层提示词；含 `engine/` 上下文装配引擎）、`memory`（事实记忆+经验记忆，引擎后端）、`knowledge`（五层知识库）、`codemap`（代码地图，改代码前查影响面）
- **工具能力**：`terminal-tools`（sandbox-exec 内核沙盒）、`desktop-tools`、`web-tools`（联网）、`office-gen`（生成 Office）、`image-gen`（生图）、`scheduler`（定时任务）、`im`（站内沟通+通知）
- **文件查看/编辑**：`excel-engine`、`pdf-viewer`、`doc-viewer`、`ppt-viewer`、`image-viewer`、`text-editor`（统一 WPS 风格壳）
- **格式解析**：`pdf/docx/pptx/xlsx/text-parser` + `image-vision`
- **样板**：`hello-world`、`_template`

## 文档规则

1. **`开发文档/` 内中文**（目录/文件名/正文）；其余目录文件名必须英文，模块 README 正文可英文（省篇幅）。
2. **一处真相，不重复**：模块细节只写在模块 README，本索引和子文档只指路。改了一处别处别复制。
3. 子文档/模块 README 写**当前真实状态 + 怎么用 + 当前规则**，不写历史流水（流水进 `变更历史.md`）。
4. 待办写 `{序号}_{任务}.md`，完成合并回 README 并删临时文档。
5. 每个模块都应有 `README.md`（接口/表/查询方式），AI 读模块先读它，不扫代码。
