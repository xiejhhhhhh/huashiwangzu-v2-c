# 上下文压缩升级方案

## 一、现状诊断

### 1.1 当前数据

| 指标 | 值 | 含义 |
|------|-----|------|
| 对话数 | 42 | |
| 事件总数 | 1,096 | agent_events 全部事件 |
| 用户消息 | 98 | agent_messages user 角色 |
| 工具调用 | 245 | tool_call 事件 |
| 压缩次数 | 7 | 仅 7 次触发过 LLM 压缩 |
| 累计折叠 | 235 | 仅折叠了 21% 的事件 |
| 硬截断 | 0 | 未发生过（好） |

### 1.2 当前机制

**触发链路：**

```
assistant 回复 → task_sink._post_turn_hooks()
  → _enqueue_context_compact()
    → SystemTaskQueue("agent_context_compact")
      → _handle_context_compact()
```

**但 90% 被跳过：** `_handle_context_compact` 第 330 行：
```python
if effective_budget > 0 and token_before <= history_budget:
    return {"status": "skipped", "reason": "within budget"}
```
预算没超 → 直接跳过。压缩只发生在**预算溢出之后**，是纯被动的。

### 1.3 压缩算法偏保守

`compressor.py` 第 18-19 行：
```python
HEAD_COUNT = 10   # 头保护 10 条
TAIL_COUNT = 20   # 尾保护 20 条
```
仅折叠中段事件。对话少于 31 条事件 → 永不触发。平均每对话 26 事件 → **多数对话永久不压缩**。

### 1.4 无预过滤

`project_to_messages()` 把所有事件类型（含 `assembly_diag`、`thinking_diag`、`understanding_diag` 等诊断事件）都投影出来。虽然 projection 阶段的 `else: i += 1` 会跳过未知类型，但诊断事件的 payload 为空字符串，不占 token。真正的问题是 **tool_result 内容过大**：单个工具输出可能几万字符，全部原封不动送进模型。

现有的 `tool_result_reducer.py` 已经做了基础裁剪（JSON 截断 + 文本首尾），但仅按字符数硬截断，缺少语义化的工具输出摘要。

**核心算法：`_find_tail_cut_by_tokens`（L2249-2351）：**
1. 从尾部反向遍历，累积每条消息的预估 token
2. 设定 `soft_ceiling = tail_token_budget * 1.5`
3. 超过 soft_ceiling 且已保条目 >= `min_tail_floor`（默认 3）→ 切割
4. 如果整段对话都在 soft_ceiling 内，用原始 budget 重走一遍
5. 锚定：确保最后一条 user_msg 和最后一条 assistant_msg 一定在保尾中
6. 对齐边界：切割点向后回退到完整的 `assistant+tool_calls` 组

## 三、升级方案

整体架构：**三级管线（Level 1→2→3 依次增强）**

```
用户提问
  │
  ▼
Level 1: 确定性裁剪（零成本，零精度损失）
  ├─ 工具结果→语义摘要   ← 参考 Hermes L490-609
  ├─ 去重相同 tool_result  ← 参考 Hermes L1062-1086
  ├─ 裁剪 tool_call arguments ← 参考 Hermes L1100-1122
  ├─ 剥离历史图片 base64  ← 参考 Hermes L1030-1050
  └─ 降低 TAIL_COUNT      ← 独立优化
  │
  ▼
Level 2: 预算驱动压缩（便宜 LLM + 滑动窗口）
  ├─ 每次回复都触发压缩（取消预算跳过检查） ← 改 _handle_context_compact
  ├─ 从固定保尾改为 token 预算保尾 ← 参考 Hermes L2249-2351
  ├─ 结构化摘要模板 ← 参考 Hermes L1502-1636
  └─ 头保护首次后衰减为 0 ← 参考 Hermes L2024-2051
  │
  ▼
Level 3: 滑动窗口压力测试（参考 Letta）
  ├─ 渐进式测试压缩比例 (30% → +10% → 符合预算) ← 参考 Letta L98-232
  ├─ 精确 token 估算（按模型选择分词器） ← 参考 Letta L27-42
  └─ 抗抖动 + 冷却机制 ← 参考 Hermes L975-984 / L1825-1826
```

def _summarize_tool_result(tool_name: str, content: str, args: dict) -> str:
    """Replace verbose tool output with a 1-line summary.

    Reference: Hermes context_compressor.py L490-609
    """
    content_len = len(content)
    line_count = content.count("\n") + 1 if content.strip() else 0

    summaries = {

        "read_file": lambda: f"[read] {args.get('path','?')} line {args.get('offset',1)} ({content_len:,} chars)",

#### 1.2 去重相同 tool_result（参考 Hermes L1062-1086）

```python
# 新增：MD5 去重
_KNOWN_HASHES: dict[str, str] = {}  # hash → "keep"

def _deduplicate_tool_results(messages: list[dict]) -> int:
    """Replace duplicate tool results with placeholder.

    Reference: Hermes context_compressor.py L1062-1086
    """
    deduped = 0
    hashes: dict = {}
    for i in range(len(messages) - 1, -1, -1):
        msg = messages[i]
        if msg.get("role") != "tool":
            continue
        content = msg.get("content") or ""
        if isinstance(content, list) or not isinstance(content, str):
            continue
        if len(content) < 200:
            continue
        h = hashlib.md5(content.encode("utf-8", errors="replace")).hexdigest()[:12]
        if h in hashes:
            msg["content"] = "[重复工具结果：内容与最近一次调用相同]"
            deduped += 1
        else:
            hashes[h] = msg.get("tool_call_id", "?")
    return deduped
```

_MAX_TOOL_ARGS_CHARS = 500

def _truncate_tool_args(args_str: str) -> str:
    """Truncate tool_call arguments JSON.

    Reference: Hermes context_compressor.py ~L1100
    """
    if len(args_str) <= _MAX_TOOL_ARGS_CHARS:
        return args_str
    try:
        args = json.loads(args_str)
        if isinstance(args, dict):
            for k, v in args.items():
                if isinstance(v, str) and len(v) > 200:
                    args[k] = v[:150] + "..."

    except (json.JSONDecodeError, ValueError):
        return args_str[:_MAX_TOOL_ARGS_CHARS] + "..."

def _strip_historical_images(msg: dict) -> dict:
    """Replace base64 images in older messages with placeholder text.

    Reference: Hermes context_compressor.py ~L1030
    """
    content = msg.get("content", "")
    if isinstance(content, list):
        new_parts = []
        for part in content:
            if isinstance(part, dict) and part.get("type") == "image_url":
                new_parts.append({"type": "text", "text": "[历史图片已剥离]"})

    return msg

#### 2.2 从固定保尾改为 token 预算保尾

**改动文件：** `modules/agent/backend/engine/compressor.py`

```python
def _find_tail_cut_by_tokens(
    messages: list[dict],
    token_budget: int,
    head_end: int,
) -> int:
    """Find tail cutoff by token budget, not fixed count.

    Reference: Hermes context_compressor.py L2249-2351
    """
    n = len(messages)
    soft_ceiling = int(token_budget * 1.5)
    min_tail = max(3, min(8, n - head_end - 1))
    accumulated = 0
    cut_idx = n

    for i in range(n - 1, head_end - 1, -1):
        msg_tokens = _estimate_msg_budget_tokens(messages[i])
        if accumulated + msg_tokens > soft_ceiling and (n - i) >= min_tail:
            break
        accumulated += msg_tokens
        cut_idx = i

    # 锚定最后一条 user 和 assistant
    cut_idx = _ensure_last_messages_in_tail(messages, cut_idx, head_end)
    return max(cut_idx, head_end + 1)
```

### Level 3：滑动窗口压力测试

#### 3.1 渐进式测试压缩比例（参考 Letta L98-232）

```python
async def _find_sliding_window_cutoff(
    messages: list[dict],
    context_window: int,
    target_ratio: float = 0.8,
) -> tuple[int, list[dict]]:
    """Iteratively find the right compression ratio.

    Reference: Letta summarizer_sliding_window.py L98-232
    """
    goal_tokens = int(context_window * target_ratio)
    eviction_pct = 0.30  # 从 30% 开始

    while eviction_pct < 1.0:
        cutoff_idx = int(eviction_pct * len(messages))
        # 找到 cutoff 前的最后一个 assistant 消息（确保完整性）
        assistant_idx = None
        for i in range(cutoff_idx, 0, -1):
            if messages[i].get("role") == "assistant":
                assistant_idx = i
                break
        if assistant_idx is None or assistant_idx >= len(messages) - 2:
            eviction_pct += 0.10
            continue

        kept = [messages[0]] + messages[assistant_idx:]
        kept_tokens = sum(estimate_one_message(m) for m in kept)
        if kept_tokens <= goal_tokens:
            return assistant_idx, messages[1:assistant_idx]  # 摘要范围
        eviction_pct += 0.10

    # 兜底：保尾 10%
    fallback_idx = int(len(messages) * 0.9)
    return fallback_idx, messages[1:fallback_idx]
```

#### 3.2 精确 token 估算

**改动文件：** `modules/agent/backend/engine/budget_allocator.py`

当前的 `estimate_one_message` 用 `len(text)/1.5` 估算，偏松。可精确化：

```python
def estimate_one_message(msg: dict) -> int:
    """Estimate tokens for a single message.

    For known models, use tiktoken. Fallback to char-based estimation.
    Reference: Letta count_tokens() L27-42
    """
    import tiktoken
    try:
        enc = tiktoken.get_encoding("cl100k_base")
        text = _extract_text(msg)
        return len(enc.encode(text)) + 4  # +4 for role overhead
    except Exception:
        return _estimate_chars_based(msg)
```

#### 3.3 抗抖动 + 冷却

参考 Hermes L975-984 / L1825-1826：

```python
# 在 ContextCompactor 类中
class ContextCompactor:
    def __init__(self):
        self._compression_count = 0
        self._last_savings_pct = 100.0
        self._ineffective_count = 0
        self._cooldown_until = 0.0

    def should_compress(self) -> bool:
        if time.monotonic() < self._cooldown_until:
            return False
        if self._ineffective_count >= 2:
            return False
        return True

    def record_result(self, savings_pct: float):
        self._compression_count += 1
        if savings_pct < 10:
            self._ineffective_count += 1
        else:
            self._ineffective_count = 0

    def record_failure(self):
        self._cooldown_until = time.monotonic() + 60
```

## 六、风险与兜底

1. **结构化摘要丢失细节** – 原始事件 append-only 保留，可通过 `compaction.folded_event_ids` 追溯
2. **便宜模型摘要失败** – 兜底为硬截断 `[摘要失败，保留头尾]`
3. **过度压缩导致缺失上下文** – 尾部 token 预算保留至少 `min_tail` 条（≥3），最后一条 user+assistant 锚定
4. **压缩太频繁** – 冷却机制（连续 2 次无效跳过 + 失败冷却 60s）
