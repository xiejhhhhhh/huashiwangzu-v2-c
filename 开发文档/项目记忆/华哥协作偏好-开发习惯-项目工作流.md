# 华哥协作偏好与项目工作流

## 核心偏好

- 项目相关工作优先主动调用项目工具台 MCP，不靠猜。
- 审计结果不能全信，必须实读和活栈验证。
- 写信给执行 agent 要落盘到投递箱，不只停留在聊天里。
- 局域网项目不需要过度安全壳，不要乱加 Docker / 强隔离。
- 比起打补丁，更倾向小范围重构来提升架构干净度。
- 文件名、目录名、任务名要语义清楚，读名大致知道内容。
- 模块边界优先，跨模块调用走统一通路，不直读别的模块表。

## 工作方式

- 项目相关关键词出现时，默认先走 brief / codegraph / routes / capabilities / db_schema / lint / probe / run_test。
- 需要发任务时，先生成投递箱信件，再交给执行 agent。
- 验收时先看回信，再自己复验命令或接口，不只看“已完成”三个字。
- 处理复杂任务时，优先收口残留，再继续扩新功能。

## 记忆用途

这条记忆用于后续华世王镞 V2 项目协作时的默认口径，作为项目层长期偏好与开发习惯基线。
