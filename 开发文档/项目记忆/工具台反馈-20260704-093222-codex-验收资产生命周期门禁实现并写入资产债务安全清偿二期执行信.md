# MCP 使用反馈

## 任务

验收资产生命周期门禁实现并写入资产债务安全清偿二期执行信

## 顺畅度

- 评分：4/5
- 体感：整体顺畅，brief/worktree_guard/code_explore/lint/run_test/call_capability/release_gate 组合能快速定位真实阻塞项。

## 本次用到的工具

brief, plan_task, worktree_guard, code_explore, Read, lint, run_test, call_capability, probe, release_gate, memory_write, mcp_feedback

## 卡点 / 不顺手的地方

call_capability 在返回较大 payload 时会截断，需要再次指定 selector 或提高 max_bytes；release_gate 输出很长，终端阅读成本高。

## 缺少的工具 / 能力

缺一个针对 release_gate JSON 自动提取 blockers/debts/counts 的小工具，避免手动从长输出里摘关键数字。

## 升级建议

release_gate MCP 返回可增加 compact_summary 字段，直接列出 verdict、clean_release_ready、blockers、debts、关键资产债务计数。

## 建议移除或合并的工具

无

## 其他备注

下一封信聚焦真实清偿与删除事件全覆盖，强调 confirm token 和误删防护。
