# 目标
把“底层全部修复好”从文件数口径改成能力域口径：每个底层域有明确验收门槛、修复批次和 release gate 结果。

# 底层域与落地顺序
1. 后台任务/worker/健康信号：先收口 `/api/health` 语义。当前运行健康必须能返回 `status=ok`；历史任务债必须暴露为 `task_queue.debt_status=debt`，但不污染 live health。24h 新增语义失败、worker 停止、event_bus 错误、module_errors 才让 health degraded。
2. codemap/锁/反馈闭环：验收 codemap feedback 空态、empirical_accuracy 不假满分、锁 ttl 参数结构化失败、manifest/capability 参数对齐、活栈锁和反馈清理。
3. knowledge pipeline：验收 pipeline debt classifier/reconcile、source missing/deleted、degraded 传播、不会 completed 包失败语义；需要 focused tests 和活栈 probe。
4. 模块注册/能力契约：用 capability_contract_diff 扫全模块，修掉可确定 drift；动态注册只允许有显式说明。
5. 数据反推与空表：空表分为正常未使用、链路已验证但无历史数据、疑似断路。疑似断路才转修复任务。
6. 全量发布门禁：最终跑 smoke/release_gate。允许历史债导致 PASS_WITH_DEBT；不允许新增 failed、24h semantic failure、health degraded、sandbox fail。

# 验收门槛
- 每个提交包必须有 focused tests、ruff（Python 改动）、必要活栈 probe/cleanup。
- 模块任务不越界；框架任务必须单独提交。
- 支线先 push；稳定后 merge main；main 再同步回支线。
