## 4. 重复造轮子清单

| 重复域 | 现状 | 收口方向 |
|---|---|---|

## 7. 可并发执行信建议

1. **Content IR 唯一化与 ParserResult Adapter 执行信**  
   目标：确认唯一 IR schema；新增 parser legacy result 到 Content IR 的 adapter；所有 parser sandbox 增加 `content:validate_ir` 断言。

2. **Knowledge 边界收口执行信**  
   目标：knowledge 不直接读写 `framework_prompt_*`、`framework_content_packages/_versions`；改为 prompt/content capability；保留 `kb_*` 自有业务表。

3. **框架任务治理去模块表执行信**  
   目标：`task_debt_governance_service` 不直接 SQL `kb_documents`；知识库任务债由 knowledge capability 报告，框架只聚合通用 task 状态。

4. **Agent/Content 产物与证据账本执行信**  
   目标：普通 Agent 对话、工具调用、生成产物默认写 workflow/verification/reference；正式产物发布到 `framework_artifacts` 和桌面文件。

5. **Sandbox 门禁补齐执行信**  
   目标：8 个正常窗口 py-only sandbox 补 Vite 六件套；后台服务 README 明确 py-only 例外；所有 README 增加 `Test data cleanup` 行。

6. **平台管理中心产品化执行信**  
   目标：用户/角色/设置/系统状态/日志/备份/反馈/公告/提示词/app-manager 合成管理类桌面应用组。

7. **模块市场与私有模块管理执行信**  
   目标：基于 `/api/private-modules/*` 做安装、预览、激活、回滚、健康检查 UI。

8. **业务模块真实交付闭环执行信**  
   目标：`wechat-writer` 补配图、排版、多版本 diff/回退、发布；`douyin-delivery` 明确是否接真实平台或只做交接包导出。

9. **视频知识库专项执行信**  
   目标：视频资产、时间片段、ASR/OCR/VLM 证据、时间点引用进入 knowledge 主链路；不与当前发布闭环混做。

## 8. 总攻大信继续做 vs 必须单独收口
