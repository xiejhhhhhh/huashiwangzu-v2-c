## 3. 总目标：一条路摸到底

最终用户路径必须尽可能闭环：

```text
用户导入/上传/桌面已有文件
→ 系统识别文件类型
→ 解析器模块产生结构化 Content IR / Knowledge 文档 / Artifact 草稿
→ Knowledge 能检索、能引用、能解释来源
→ Agent 能基于文件/知识/Artifact 做任务
→ Agent 工具调用、子代理、失败、引用、产物全可见
→ ContentPackage 能预览、发布 Artifact/File
→ 桌面能看到、打开、下载、回源、删除/恢复
→ 任务中心/通知中心/反馈中心能展示过程与可操作项
→ ReleaseGate/Smoke/Test 能判定真实可发布状态
```

不是只让接口通过，而是要让普通用户在桌面上能理解“发生了什么、失败在哪里、下一步能点什么”。
