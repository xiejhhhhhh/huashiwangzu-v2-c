- `backend/app/routers/system_status.py:13` — `GET /api/system/status` 无任何 auth
- `backend/app/routers/system_status.py:19` — `GET /api/system/trace/{trace_id}` 无任何 auth
- **影响**: 任何人可读系统状态、DB 元数据、全部分布式追踪树

### S2. 硬编码 API 密钥在 .env（磁盘明文，已在 .gitignore 中）
- `backend/.env` 包含 6 个实时 API 密钥/凭据：
  - `JWT_SECRET=huashi-wangzu-v2-secret-key-2026`（弱）
  - `DEEPSEEK_API_KEY`、`GPTSTORE_API_KEY`、`LIBLIB_ACCESS_KEY`、`LIBLIB_SECRET_KEY`、`MIMO_GATE1_KEY`
  - `DB_PASSWORD=123rgE123`
- **状态**: `.env` 已在 `.gitignore` 第 16 行配置，`git log` 确认**从未被提交到版本控制**。但磁盘上仍明文存储，有本地泄露风险。
- **影响**: 攻击者若获得本地文件访问权限可消耗 LLM 配额、解码 JWT、直连 DB
- **修复**: 确认 `.gitignore` 已有 `.env`（已存在）；轮换所有密钥；改用环境变量注入，去除 `.env` 文件依赖

### S3. 12 个模块 `/health` 端点无任何认证
- agent、desktop-tools、web-tools、terminal-tools、knowledge、image-gen、image-vision、office-gen、excel-engine、pdf-parser、docx-parser、pptx-parser、text-parser、xlsx-parser 的 `GET */health` 全部无 auth

### S4. `/api/files/share/*` 7 个端点仅 `get_current_user`，无角色检查
- `backend/app/routers/file_shares.py:59-158`
- **影响**: 任意认证用户（含 viewer）可创建/更新/删除文件分享

### S5. vector_search() 全表加载到 Python 做余弦相似度
- `modules/knowledge/backend/services/search_service.py:189-212`
- 加载所有 `owner_id + embedding is not null` 的 chunk 到 Python，逐条计算余弦相似度
- **影响**: 10 万条 chunk 时每次搜索全量加载，内存和 CPU OOM
- **修复**: 改用 pgvector SQL 操作符 `ORDER BY embedding <=> :query_emb`

### S6. 6 个框架模型未注册到 Alembic env.py
- `FileAsset`、`FileUploadSession`、`WorkflowDefinition`、`WorkflowRunRecord`、`WorkflowStepRecord`、`SystemTraceSpan`

### S7. 全代码库零次 `selectinload`/`joinedload` 使用
- 所有关系访问（如 `FileJsonPackage.versions`）触发额外 SQL
- **影响**: 大量 N+1 查询未被框架发现
