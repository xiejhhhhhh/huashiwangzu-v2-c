## 本轮只做 3 个目标

### 目标 1：删除事件不漏

修复文件永久删除链路，保证所有真实被永久删除的文件都会发 `file.permanent_deleted`。

必须覆盖：

1. 单文件永久删除。
2. 文件夹永久删除时的所有子文件。
3. 清空回收站时的顶层文件 + 文件夹内所有子文件。
4. desktop-tools 删除路径：确认是否复用框架删除 API；如果绕过，补 `file.deleted` / `file.permanent_deleted`。

验收标准：

```text
永久删除单 file -> 1 个 file.permanent_deleted
永久删除 folder -> 子文件全部有 file.permanent_deleted
empty_trash -> 顶层 file 与 folder 子 file 全部有 file.permanent_deleted
desktop-tools 删除路径不绕过生命周期事件
```

实现建议：

- 让 `recycle_service.delete_permanently` / `empty_trash` 返回 `permanently_deleted_file_ids`。
- router 根据返回 ids 发事件。
- 不要让 service 层硬 import event bus。

### 目标 2：历史资产债务真实清偿

把已经确认源文件不可用的历史债务安全归档/清理。必须 dry-run 先行，真实执行必须 confirm token。

#### 2.1 ContentPackage

新增能力：

```text
content:archive_lifecycle_unavailable_packages
```

参数：

```json
{
  "dry_run": true,
  "limit": 100,
  "reason": "source_unavailable | source_file_deleted | source_file_missing",
  "confirm": "ARCHIVE_LIFECYCLE_UNAVAILABLE_PACKAGES",
  "audit_reason": "string"
}
```

真实执行时：

```text
status = archived
parse_error = source_file_deleted/source_file_missing
manifest.lifecycle.archived_by_lifecycle = true
manifest.lifecycle.source_lifecycle_state = source_recycled/source_missing
manifest.lifecycle.previous_status 保留
不删除 package/version/resource
```

同时修复 `missing_current_version` 口径：

- `archived_by_lifecycle`、`pending`、`failed` 不应因为没有 current_version 直接 BLOCKER。
- parsed/degraded/partial 且应有内容版本但 current_version 空，才是 BLOCKER。
- 如有历史 version 可回填，新增或实现修复能力：`content:repair_missing_current_versions`，真实执行必须 `confirm="REPAIR_CONTENT_CURRENT_VERSION"`。

#### 2.2 Knowledge

使用已有能力：

```text
knowledge:archive_source_unavailable_documents
confirm="ARCHIVE_SOURCE_UNAVAILABLE"
```

只归档源文件 deleted/missing 的历史债务：

```text
kb_documents.deleted = true
保留 chunks/page_fusions/raw_data，不做物理删除
```

#### 2.3 Test pollution

收紧测试 marker，避免误删真实用户文件。

不要再把任意 `test-` 作为强 marker。建议强 marker：

```text
smoke-
e2e-
recycle-
pytest-
test-upload-
test-file-
test-pollution-
lifecycle-source-
permanent-source-
```

真实清理必须 `confirm="CLEAN_TEST_DATA"`，并按顺序：

```text
归档关联 Knowledge
归档关联 ContentPackage
删除 recycle rows
删除 file rows
物理文件只允许删除 upload root 内、且来自候选 file.storage_path 的文件
```

验收标准：

```text
Knowledge source_unavailable: 168 -> 0 或全部进入明确 archived 治理态
ContentPackage source_unavailable: 204 -> 0 或全部 archived_by_lifecycle
ContentPackage missing_current_version: 2 -> 0 或不再误判 BLOCKER
active marker test files: 4 -> 0
recycled marker test files: 306 -> 0
knowledge_documents_from_test_files: 170 -> 0
content_packages_from_test_files: 207 -> 0
```

### 目标 3：release gate 语义稳定

清偿后，release gate 必须能清楚表达：

```text
BLOCKER -> release_safe=false, deploy_allowed=false, clean_release_ready=false
PASS_WITH_DEBT -> release_safe=true, deploy_allowed=true, clean_release_ready=false
PASS(full UI) -> release_safe=true, deploy_allowed=true, clean_release_ready=true
preflight/skip-ui 永远不能 clean_release_ready=true
```

本轮最低验收：

```text
release_gate(skip_ui=true, mode=preflight)
不得再因为 ContentPackage lifecycle debt 或 Test data pollution 返回 BLOCKER
```

如果仍有其它 BLOCKER，必须如实列出，不要降级粉饰。
