# V1 能力对照与 V2 用户路径缺口报告

调研时间：2026-07-04  
执行 agent：codex-v1-v2-user-path-gap  
任务来源：`开发文档/项目记忆/调研信-V1能力对照与V2用户路径缺口.md`

## 结论摘要

V2 不是 V1 的功能搬运，而是把 V1 的桌面壳、文件、知识库、Agent、Office、权限等能力拆成 `frontend/ + backend/ + modules/` 的干净架构。底层能力总体已经超过 V1：模块 manifest、能力注册表、跨模块调用、Content IR/Artifact、Agent workflow、知识库主链路、文件权限和 release gate 都已建立。

当前最明显的用户路径缺口不是“有没有 API”，而是“用户今天从桌面入口能不能顺手完成”。P0 级缺口集中在可见工作台和端到端产品闭环：管理中心缺桌面入口组、Office/Content IR 缺 V1 Office 工作台式的可视化版本/补丁/导出闭环、Agent 产物与失败恢复还未默认沉淀为用户可回放账本、模块市场/私有模块管理有 API 但缺 UI。

## P0 用户路径缺口

| 缺口 | 今天用户会怎么卡住 | 已有底座 | 建议进入哪封大信 |
|---|---|---|---|
| 平台管理中心缺桌面入口组 | 管理员要管理用户、角色、系统配置、应用状态时找不到 V1 那类清晰入口 | `/api/users`、`/api/roles/matrix`、`/api/settings/system-config`、`/api/app-manager/apps` | 平台管理中心产品化大信 |
| Office/Content 工作台缺可视化闭环 | 用户能生成样例、Agent 能写 IR，但看版本、预览 diff、发布/导出/替换原文件不顺手 | ContentPackage、Artifact、content routes、office-gen、excel-engine | Office/Content 工作台与文档编辑器产品化大信 |
| Agent 产物/证据/失败恢复未默认成用户账本 | 用户让 Agent 做长任务，失败、产物、验证和来源不一定形成可回放任务卡 | Agent workflow 表/API、EvidenceReference、Artifact | Agent/Content 产物与证据账本大信 |
| 我的任务/后台任务中心缺用户入口 | 后台任务、知识库分析、Agent workflow、失败重试分散在通知/日志/API | task queue、worker audit、Agent workflow、taskbar notification | 任务中心与通知中心大信 |

## 已替代能力

|---|---|
| 中文桌面应用注册表 + DB 同步 | 英文 manifest + backend sync + component key map + background-service 契约 |
| 文件 JSON | Content IR + ContentPackage + ContentPackageVersion + Resource/ResourceRef |
| Office 导出 | office-gen + ContentExportService + Artifact + excel-engine compile |
| 静态 Agent 工具列表 | capability registry + `skill_list/skill_describe/skill_use` 渐进式工具发现 |

## 不应恢复的 V1 包袱

| 包袱 | 原因 |
|---|---|
| Laravel/PHP 旧控制器、旧路由、旧 Worker 脚本 | V2 是 FastAPI/Vue/modules 干净架构，不能混入旧树 |
| 中文源码目录/API 路径 | V2 规则要求开发文档外英文命名；中文只做 UI/文档展示 |
| 传统后台菜单体系 | V2 产品方向是桌面壳 + 应用注册，不回退到左侧菜单后台 |
| 文件 JSON 旧补丁模型 | 能力窄且耦合 Office；V2 Content IR/Artifact 更适合作为统一源 |
| AI 工具箱作为普通用户测试台 | V2 应把工具治理放 Agent admin/技能治理，不把测试控制台当产品主入口 |
| 模块直接互相 import 或读表 | V2 跨模块必须经框架统一能力通路 |

## 后续大信建议

1. **平台管理中心产品化大信**  
   范围：用户管理、角色权限矩阵、系统配置、系统状态、系统日志、备份状态、应用管理器。原则：一个管理中心桌面 app 组，不恢复 V1 散落小后台。

2. **Office/Content 工作台与文档编辑器产品化大信**  
   范围：ContentPackage/Artifact 列表、版本历史、block diff、资源查看、发布到桌面文件、导出、替换原文件、Excel 状态对齐、Word/PPT 先做块级编辑。

   范围：普通 Agent 对话和长任务默认生成 workflow、tool ledger、verification、artifact/reference；失败可重试、取消、继续、导出错误报告。

4. **任务中心 + 通知/反馈中心大信**  
   范围：我的任务、后台任务、知识库分析任务、Agent workflow、scheduler、IM 系统通知、公告、内测反馈状态流转。

5. **模块市场与私有模块管理大信**

   范围：视频/音频上传、ASR segment、关键帧 OCR/VLM、时间点引用、模型 key/上下文预算、降级可见。

8. **业务模块真实交付闭环大信**  
   范围：wechat-writer 配图/排版/diff/发布，douyin-delivery 真实平台或交接包导出，产物统一进 Artifact/File。
