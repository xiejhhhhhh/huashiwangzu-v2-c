# 收口定义
本轮底层收口不是无限扫雷，而是满足发布门禁：当前 health/system status 绿、队列新增失败不增长、能力契约无阻塞漂移、当前 dirty 大域 focused tests 通过、sandbox matrix 0 fail、release_gate --skip-ui 无 BLOCKER。

# 硬门槛
- /api/health: status=ok, database=ok, module_errors=null, worker.running=true, event_bus=ok, event_bus_stuck_processing=0, semantic_failed_completed_24h=0。
- /api/system/status: backend/database/worker 必须 true；本次实测 model_service/entry 也 true。
- scripts/check-capability-drift.py: 必须 OK；本次实测 OK (170 registered public capabilities)。
- dev_toolkit capability_contract_diff: 可确定 manifest/runtime 参数漂移应清零；动态注册站点需说明。
- module sandbox matrix: 0 fail，skip 作为 DEBT。
- release_gate --skip-ui: 允许 PASS_WITH_DEBT，不允许 BLOCKER。

# 当前缺口清单
1. codemap/锁/反馈闭环：实现已落地并通过 83 tests + 活栈 stats/list_feedback；剩一个 contract diff：report_inaccuracy manifest 有 agent_id 但 runtime 无 agent_id。根因=契约元数据/handler 参数兼容，属于健壮性补线。
2. knowledge pipeline：dirty 涉及 document_service、pipeline_debt_service、pipeline_orchestrator、pipeline_service、stage tests；需等 worker/主会话验收。根因待分：pipeline 状态归属/历史债分类，可能是架构语义补齐而非简单 bug。
3. agent contract drift：7 个 manifest 参数少于 runtime。根因=manifest 元数据滞后；需 contract lane 修模块 manifest，不碰框架。
4. desktop-tools contract drift：list_files/search_files runtime 参数元数据为空。根因=register_capability 元数据不足或 manifest 声明过细；需按真实 handler 对齐。
5. image-gen usage_history：manifest 缺 limit。根因=manifest 元数据滞后。
6. 项目记忆临时节点：codemap/douyin/image-gen 等临时节点可跟提交进入记忆；长期文档蒸馏可后续单独做，不阻塞本轮底层 release gate。

# 执行顺序
先收 codemap（含 agent_id drift 小修）→ 收 knowledge → 收 contract drift worker 结果或派专项 → 跑 sandbox matrix/release_gate → 合 main。

# 停止条件
当上述硬门槛全部满足，且 release_gate --skip-ui 为 PASS/PASS_WITH_DEBT，停止本轮底层扫雷并合并主线；新发现但非阻塞的历史债进入后续队列。
