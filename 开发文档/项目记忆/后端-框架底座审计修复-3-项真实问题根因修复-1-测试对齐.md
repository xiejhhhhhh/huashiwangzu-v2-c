# 改了什么

1. **content.py trigger_pipeline REST 端点返回 200+success=false → 抛 AppException(422)**：违反"禁止 return ApiResponse(success=False) 返回 200"契约。改为 raise AppException(status_code=422)，错误语义正确由统一异常处理返回 `{success:false, data:null, error:"..."}`。

2. **private_module_service.py capability 清理错配**：`_unregister_private_module` 原来用 `unregister_capability(private_key)` 清理，但 `private_key = f"__private__{owner_id}__{module_key}"` 不匹配模块实际注册能力（模块用 manifest key）。这导致能力泄露。
   - `module_registry.py` 新增 `_current_capability_keys()` 快照函数
   - `private_module_service.py` 增加 `_PRIVATE_MODULE_CAP_KEYS` 追踪映射
   - `activate_private_module` 激活前后快照差异，记录新增能力 key
   - `_unregister_private_module` 用已追踪的实际 key 精准清理，不再传错 key

3. **deactivate_private_module 无失败保护**：`_unregister_private_module` 异常会传播导致 DB 与运行时状态不一致。改为 try/except 包裹，失败时 raise AppException(500) 不提交 DB 状态变更。

4. **测试对齐**：`test_pipeline_rest_propagates_failed_result` 原断言 `result.success is False` (200 响应)，改为 `pytest.raises(AppException)` 验证 422 异常语义。
