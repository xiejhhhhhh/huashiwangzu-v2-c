# MCP 使用反馈

## 任务

多子代理底层维修整合、验证、准备提交推送

## 顺畅度

- 评分：4/5
- 体感：整体顺畅，工具台帮助快速定位契约/活系统状态；命令行 release_gate 已可用。

## 本次用到的工具

brief, plan_task, worktree_guard, codegraph/code_node/code_impact, probe, capabilities, routes, finish_task, memory_write, mcp_feedback, release_gate

## 卡点 / 不顺手的地方

MCP 进程可能持有旧 release_gate 代码，曾与命令行结果不一致；多 sandbox 同名 test_module.py 合跑会 pytest import mismatch，需要矩阵工具逐模块跑。

## 缺少的工具 / 能力

建议新增 durable agent board/claim heartbeat 工具；建议 release_gate MCP 工具能显示实际入口版本/hash。

## 升级建议

把 sandbox matrix 的逐模块隔离机制暴露为 run_test 多目标模式，避免同名测试模块导入冲突；给 release_gate 增加队列历史债分项治理建议。

无
