## 1. 结论摘要

- 当前产品化成熟度：6.1/10
- 桌面体验成熟度：6.2/10
- 测试发布成熟度：6.8/10
- 当前判断：华世王镞 V2 已经有“能跑的桌面壳 + 模块能力 + 后台队列 + 发布门禁”，但还不是“普通用户能顺手完成一件事”的产品。主要差距不在单个 API 是否存在，而在“入口、过程、反馈、产物、下一步、历史追溯”没有合成一个闭环。

最值得下一步投入的 3 件事：

1. 把“任务/通知/Agent 工作流”合成桌面级反馈中心：任务栏能看到后台正在做、失败了什么、是否待确认、产物在哪里。
2. 把文件和知识库主链路做成用户闭环：上传后明确显示解析状态、失败原因、可重试动作、问 AI、导出和生成文档。
3. 把 release gate 固化成分层门禁：quick 30-60 秒、module 1-3 分钟、preflight 3-5 分钟、full regression 可更久但可解释。

## 5. 桌面体验审计

成熟度评分：6.2/10。

已经完成的部分：

- 桌面壳有窗口系统、任务栏、启动器、右键菜单、文件图标、文件管理器、回收站。
- `CommandRegistry` 已存在，能注册 app/action 并搜索。
- 文件打开调度规则清楚：`editable_formats` 优先，`supported_formats` 次之，`sort_order` 排序，不对 `["*"]` 做误匹配。
- Agent 内部已有“正在工作/已工作”、tool cards、workflow list/detail 的 UI 方向。
- Playwright 已覆盖 storageState 登录复用，说明桌面 UI 曾被系统性验证。

主要判断：

桌面壳现在像“能打开应用的壳”，还不像“能交代工作进展的办公桌面”。用户完成真实任务时，最重要的信息不是窗口数量，而是：后台有没有在跑、哪里失败了、是否需要我确认、产物在哪里、下一步做什么。

## 7. 测试分层与发布效率审计

成熟度评分：6.8/10。

已经完成的部分：

- `release_gate` 有机器可读 JSON，区分 `clean_pass`、`release_safe`、`has_debt`。
- `release_gate(full, skip_ui=true, sandbox_jobs=4, sandbox_frontend_jobs=2)` 本次 69.225 秒完成，后端 smoke 28/29，sandbox 35/35。
- `module_sandbox_matrix` 已经从早期大量 skip 进步到 35/35 pass。
- `smoke` 会关注异步队列前后水位，避免只看 HTTP 200 假绿。
- Playwright 有 `globalSetup` 和 storageState，已经避免每测重登。

主要问题：

- pytest 默认配置只注册 marker，缺 `testpaths/addopts` 和默认层级选择。
- 有重复 `pytest.ini` 和 `backend/pytest.ini`。
- 子代理只读收集发现 `pytest --collect-only` 可收集到 1240 tests，但在 `modules/codemap/tests/test_feedback_capabilities.py` 触发 SQLAlchemy 表重复定义错误，说明全量收集本身不稳定。
- Playwright 配置 `workers: 1`，主 spec 仍有 serial，UI full gate 会被拖慢。
- 默认 Playwright 仍可能起 Vite；虽然支持 `PLAYWRIGHT_EXTERNAL_SERVER=1`，但默认不完全贴合“活栈常驻夹具”。
- UI 测试曾写报告到邮箱/收件箱，容易产生持久副作用，不适合 release gate。

推荐测试分层：

| 层级 | 目标耗时 | 内容 | 结论口径 |
|---|---:|---|---|
| quick check | 30-60 秒 | `release_gate(mode=preflight, skip_ui=true)` + 受影响文件 `run_test`/lint | 只判断当前是否可继续开发，不宣称 clean release |
| module check | 1-3 分钟 | 当前模块 sandbox、相关 pytest、前端 build 或局部 UI project | 判断模块任务是否可合入 |
| preflight release gate | 3-5 分钟 | backend smoke、任务队列 delta、sandbox matrix 并发、dirty 摘要 | 可给 `PASS_WITH_DEBT`，必须列债 |
| full regression | 可更久 | 完整 release gate，不跳 UI；UI 拆 project 并发；失败按模块归因 | 只有这里能给 clean release |
