# 前端类型安全与假空态全局清扫收口

## 结果

- 范围：`frontend/src`、`frontend/tests`、`modules/*/frontend`，并顺手修正 `modules/agent/sandbox/src/App.vue` 的唯一 `any`。
- 类型压制清理：`any/as any/@ts-ignore/@ts-expect-error` 扫描从 1 处降为 0 处；最终扫描无命中。
- 构建验收：`npm --prefix frontend run build` 通过。

## 已覆盖页面与入口

- 主壳 API 错误：`ApiResponse.errors` 对齐后端字典形态，`toApiErrorInfo()` 补充 `errors` 与 FastAPI `detail[]` 摘要。
- 桌面应用注册：空权限不再误判为禁止所有用户，空应用列表不再伪装成注册表加载失败。
- 通知中心：LoadState 错误条新增复制错误、重试、跳转来源；知识库状态返回缺失 `stuck_documents` 时先归一化再进入 ready。
- Agent：对话消息加载失败不再显示“开始对话吧”，改为可重试错误条；工具结果图片解析改为类型守卫。
- Knowledge：文件树区分 loading/error/empty；详情页画像、逐页内容、关联数据失败时显示 pane error；健康看板接口失败显示错误摘要与重试。
- IM：会话、消息、联系人加载/发送失败从 `console.error` 升级为可见错误条，不再显示假空会话。
- Douyin Delivery：修正模块 API base 从 `/api/douyin-delivery` 到 `/douyin-delivery`，避免 runtime 生成 `/api/api/...`。

## 后续补项

- Playwright 仍缺通知中心失败态专项：建议 mock 一个反馈端点 503，断言错误条、复制、来源跳转与重试。
- Agent `WorkflowDetail.vue` 的可选摘要/ledger 仍有 `catch -> []` 的局部假空态，建议按 section 拆状态。
- Knowledge `WorkspaceGraph.vue` 的图谱失败仍可能落到“尚无关联数据”，建议补 graph load state 与 relation fallback。
- WeChat Writer、Image Gen、Office Gen 的加载失败多为 toast 或静默降级，建议下一轮补持久错误条。
