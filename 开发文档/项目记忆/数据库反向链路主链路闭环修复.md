# 改了什么

- 收紧 `framework_system_task_queues` 的任务操作状态机：`retry` 只允许 `failed`，并清理旧 `result/started_at/completed_at/error_message/retry_count`；`cancel` 只允许 `pending/running`。
- 收紧通知反向链路：`/api/notifications/module` 从 viewer 提升为 admin，避免普通模块/用户随意创建系统公告；新增权限测试。
- 前端通知面板补任务债务摘要入口：打开任务栏通知面板时读取 `/api/tasks/worker/audit`，展示后台任务失败/待处理/运行中和债务状态；通知类型字段改为后端真实 `notification_type`。
- dev_toolkit 配置迁移：删除带本地凭据风险的 tracked `dev_toolkit/config.json`，新增安全示例 `dev_toolkit/config.example.json`、本地 `config.local.json` 忽略规则和 `config_loader.py`，支持环境变量覆盖。
- release gate 输出增强：机器 JSON 包含 git sha/branch/dirty、backend/frontend 可用性、health/system status、任务债务摘要，并支持 sandbox jobs/frontend jobs 透传。
- 修复知识库 `chunking_service.py` 的 `nonlocal chunk_index` 语法阻断，恢复 `/api/health` 为 ok。
