# 变更历史（里程碑 + 关键踩坑）

## 桌面窗口累积爆炸修复——去重+上限+恢复替换（2026-06-21）

- **根因**：`openWindow('desktop',...)` 对 `allowMultiple=true` 的 desktop 每次 push 新窗口，不做文件夹去重，持久化后 restore 翻倍，最终 16382 个窗口耗尽浏览器连接。
- **防线 1 同文件夹去重**：`openWindow` 对 desktop 按 `(payload?.folderId ?? null)` 查重，命中则 focus 旧窗口，不新建。
- **防线 2 恢复替换**：`restoreWindows` 开头 `windows.splice(0, windows.length)` 清空再恢复，杜绝多次 restore 翻倍。
- **防线 3 硬上限 30**：`MAX_WINDOWS = 30` + `deduplicateSnapshots()` 在保存和恢复两路都做去重截断；`openWindow` 末尾 `windows.length >= 30` 时拒绝。
- **改了**：`frontend/src/desktop/window-manager/` 下 4 文件（window-manager.ts / desktop-session-storage.ts / desktop-session-restore.ts / desktop-state-store.ts）
- **验证**：Playwright 真浏览器注入验证——同文件夹窗口 ID 相同(去重✅)、不同文件夹新建(正常✅)、30 限拒绝超额(封顶✅)、三次 restore 不翻倍(替换✅)、npm run build ✅

## 知识库五层多轮交叉印证 + 跨文件动态关联（2026-06-21，真实业务 PDF 验收通过）

按华哥 V1 真实设计落地（不是简化版"抽文本→分块→向量"）。全在 `modules/knowledge/`，视觉/嵌入/LLM 走框架网关。
- **第1层 原始多轮采集**（`raw_collection_service`/`pdf_render_service`，落 `kb_raw_data`，落盘后不可变）：第1轮文本提取 / 第2轮截图OCR（fitz渲染PNG→VLM）/ 第3轮视觉构成（VLM描述版面）。三轮独立证据分别存，防单一信息源偏差。
- **第4层 页级融合**（`fusion_service`，落 `kb_page_fusions`）：三轮 LLM 交叉印证 → 带 `融合正文/置信度/冲突JSON` 的单页权威描述。**对外检索默认用这层**。
- **第5层 文件画像**（`profile_service`，`kb_document_profiles`）：N页串联 → 主旨/摘要/章节/关键实体 + 画像向量(1024维bge-m3)。
- **第6层 实体图谱**（`entity_service`，从融合层重建）。
- **第7层 跨文件动态关联**（`relation_service`，`kb_file_relations`）：新文件增量与已有文件织网，画像向量余弦(0.6)+实体共现Jaccard(0.4)。华哥最看重。
- **全链路管道**（`pipeline_service`，`kb_pipeline`）：采集→融合→画像→图谱→关联，每步 commit，幂等可重入。

**派生层全部后台任务化 + 逐项提交**：原只有采集异步，融合/画像/关联/图谱都是同步端点→6页就 60s 超时、末尾才 commit→超时回滚数据全丢。改成入队 `kb_fuse`/`kb_profile`/`kb_relation`/`kb_graph`/`kb_pipeline`，由 `knowledge_worker` 消费；逐页/逐边 commit，中断只丢当前项、重跑跳过已完成。

**验收（真实微盘 PDF，华哥亲验路线 doc3红绷带/doc4圣痘士/doc5白富美）**：三份全链路 done，融合带置信度(0.87-0.94)+冲突记录，画像内容准确，图谱8节点8边，跨文件关联6条边互相织网；检索召回融合正文、Agent `knowledge:search` 闭环通；重启后 doc5 管道幂等续跑成功。pytest 81 passed。

**坑（真实数据验证才挖出，自测全过但功能崩）**：
- 报告自报"全链路通过"实为只用纯文本文件跑了简化老链路，五层灵魂一行真数据没产出。**纯文本文件无真图，第2/3轮 base64 把文本当 PNG 发给 VLM → 400**；真 PDF 渲染真 PNG VLM 正常。
- **派生层同步超时丢数据**：末尾一把梭 commit，超时杀连接→事务回滚→0行。改逐项 commit。
- **检索没接融合层**：`hybrid_search` 只查 `kb_chunks`，融合层(`kb_page_fusions`)无 embedding 列、走五层管道的文档检索 0 命中。修：融合落库后把"融合正文"切块+嵌入写进 `kb_chunks`(`index_fusions_to_chunks`)，检索主干不动即召回融合内容。
- `rebuild-graph` 端点遗漏未异步化(改派生层时漏改)→仍 60s 超时；补入队 `kb_graph`。
