# 明确文件平台能力和模块业务边界

## 当前发现

- 文件相关后端代码散落在 `backend/app/routers/files.py`、`backend/app/routers/file_transfer.py`、`backend/app/routers/recycle.py` 和多个 service 文件中。
- 平台能力（存储、权限、上传下载）和业务流程（界面、操作流）没有明确分离。
- 当前没有模块 runtime 来提供 API 根路径和权限配置。

## 目标

- `backend/` 提供文件存储、权限、上传下载、预览能力。
- `modules/file-manager/` 提供文件管理界面和业务流程。
- 模块通过 runtime 获取 API 根路径和权限能力。

## 验收

- 文件管理模块不直接写死平台路径。
- 测试上传文件用完会清理。

完成后把当前文件模块边界合并回本目录 `README.md`，再删除本文件。
