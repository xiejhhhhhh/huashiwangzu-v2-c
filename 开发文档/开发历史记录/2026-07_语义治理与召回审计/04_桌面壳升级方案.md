# 桌面壳升级方案

## 问题发现（2026-07-14）

华哥对标Windows桌面还原度审计后，确认以下核心问题：

### 一、底座架构问题

当前桌面壳代码组织：
```
frontend/src/desktop/
├── shell/              壳主体（index.vue 373行调度层）
├── window-manager/     窗口管理（window-manager.ts 227行）
├── taskbar/            任务栏
├── launcher/           启动器
├── app-registry/       应用注册
├── context-menu/       右键菜单
├── clipboard/          剪贴板
├── drag-drop/          拖拽
├── selection/          框选
├── events/             事件总线
└── design-system/      设计令牌
```

**底座疑问**：
1. window-manager.ts 只有227行，窗口状态用 reactive 数组管理——能撑住复杂动画和状态同步吗？
2. 图标位置持久化依赖 DOM 查询 + transform，没有独立的网格模型
3. 壁纸、图标大小、布局方式全硬编码，无配置系统
4. 动画完全靠 CSS transition，无帧控制能力

### 二、拖拽系统问题

- drag-state.ts 用 DOM 操作（设 opacity/pointerEvents）管理拖拽中状态
- 没有逻辑网格，释放位置靠真实像素 transform 记忆
- 拖入文件夹判定靠 data-folder 属性轮询，没有碰撞检测模型
- 多选拖拽时 offsetList 算法依赖 getBoundingClientRect，窗口滚动/缩放时会偏

### 三、用户反馈感知问题

- 窗口打开：直接 push 到数组 → DOM 渲染 → CSS动画。无加载中间态
- 操作无声音/震动/微动画反馈
- 长操作（文件上传等）在任务栏无进度条
- 右键菜单执行命令后无即时视觉确认

---

## 解决方案

### 方案总纲：底座可配置化 + 微交互补全 + 拖拽重写

#### A. 底座配置系统（新增 desktop-config）

新建 `frontend/src/desktop/config/` 目录：

| 文件 | 职责 |
|------|------|
| desktop-preferences.ts | 用户偏好状态（壁纸/图标大小/网格间距/布局方向）|
| icon-layout-config.ts | 图标网格模型（行列计算、吸附点、间距） |
| window-defaults.ts | 窗口默认行为配置（动画时长/贴靠阈值/最小尺寸） |
| taskbar-config.ts | 任务栏配置（位置/高度/合并策略/时钟格式） |

偏好数据存 localStorage + 后端同步（已有 desktop API），用户改完即生效。

#### B. 图标网格模型（重写 drag-drop）

当前：自由像素定位 → 改为：**逻辑网格坐标系**

```
网格模型：
- 计算桌面可用区域（去掉任务栏高度）
- 按图标尺寸(可配置) + 间距(可配置) 算出行列数
- 每个图标占一个格子，坐标是 (row, col)
- 渲染时 row/col → 像素位置（纯计算，不查DOM）
- 拖拽释放 → 找最近空格 → 吸附
- 持久化存 {图标key: [row, col]} 而非像素transform
```

#### C. 窗口动画升级

| 动作 | 当前 | 升级后 |
|------|------|--------|
| 打开 | translateY(10px)+opacity | 从图标位置缩放展开（记住来源坐标） |
| 关闭 | opacity→0 | 缩小回来源位置或任务栏位置 |
| 最小化 | opacity→0 | 缩小飞入任务栏对应按钮位置 |
| 最大化 | 直接改尺寸 | 平滑展开（border-radius渐变为0） |
| 还原 | 直接改尺寸 | 平滑收缩（border-radius恢复） |
| 贴靠 | 有预览但无过渡 | 窗口平滑滑入贴靠位置 |

**最大化拖拽还原**：监听标题栏mousedown → 如果当前maximized → 先还原到鼠标比例位置 → 进入拖拽模式

#### D. 任务栏增强

1. **悬停预览**：鼠标悬停任务栏项 300ms 后弹出缩略图（用窗口内容区截图或实时渲染小窗）
2. **右端"显示桌面"窄条**：12px宽透明区域，hover显示竖线，点击全部最小化/还原
3. **时钟升级**：显示日期，悬停弹完整时间面板
4. **进度条**：任务栏按钮底部可显示彩色进度条（通过事件总线传入进度百分比）

#### E. 启动器重做

- 从左下角窄面板 → 底部居中大面板（480px宽）
- 上半部分：已固定应用网格（4~5列）
- 下半部分：最近打开的文件/推荐
- 顶部搜索栏保留
- 底部：用户头像 + 登出

#### F. 全局反馈机制

新增 `frontend/src/desktop/feedback/` 目录：

| 文件 | 职责 |
|------|------|
| micro-animation.ts | 微动画库（弹跳/脉冲/闪烁/涟漪） |
| operation-toast.ts | 操作轻提示（右下角快速消息+图标） |
| progress-bus.ts | 进度事件总线（任务栏进度条数据源） |

原则：**用户每个操作都有即时视觉响应**
- 点击图标：图标弹跳一下
- 打开窗口：有展开动画
- 拖拽图标：拖动中有半透明幽灵+目标格高亮
- 关闭窗口：缩小消失而非瞬间消失
- 文件操作成功/失败：右下角微提示
